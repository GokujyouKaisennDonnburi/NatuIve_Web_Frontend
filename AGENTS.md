# AGENTS.md

# AI開発ガイドライン

このドキュメントは、AIエージェント（OpenCode、Claude Code、GitHub Copilot Agent、Cursorなど）が本プロジェクトで開発を行う際の共通ルールです。

## 目的

本プロジェクトでは、AIによるコード生成・リファクタリング・レビュー・ドキュメント作成を活用しています。

AIは新しい実装方法を提案することよりも、**既存実装との一貫性を維持すること**を最優先としてください。

---

# 基本方針

## 1. 既存実装を最優先する

新しいコードを書く前に、必ず類似する実装を探してください。

以下の優先順位で確認してください。

1. 同様の画面
2. 同様のコンポーネント
3. 同様のAPI呼び出し
4. 同様のHook
5. 同様のユーティリティ

既存実装に合わせられる場合、新しい設計を導入しないでください。

---

## 2. プロジェクトのルールを遵守する

実装時は以下のドキュメントを参照してください。

- docs/architecture.md
- docs/coding-rules.md

また、Claude Code を利用する場合は CLAUDE.md に記載されたルールも遵守してください。

---

## 3. 一貫性を最優先する

より良い実装と思われる場合でも、

- 命名規則
- コンポーネント構成
- ディレクトリ構成
- 実装スタイル

は既存コードに合わせてください。

プロジェクト内で統一されている実装を優先してください。

---

# 実装時のルール

AIは以下を必ず確認してください。

- 同様の画面が存在しないか
- 再利用可能なコンポーネントが存在しないか
- 同じ責務のHookが存在しないか
- servicesの実装方法
- 型定義の配置場所
- Tailwindの記述方法

既存の実装を流用できる場合は、新規作成よりも再利用を優先してください。

---

# コードレビュー時

レビューでは以下の観点を確認してください。

- 型安全
- 命名規則
- コンポーネントの責務
- 再利用性
- 可読性
- パフォーマンス
- アクセシビリティ
- レスポンシブ対応
- 不要なコード
- デッドコード
- エラーハンドリング

改善案がある場合は、理由も合わせて説明してください。

---

# リファクタリング

リファクタリングでは以下を守ってください。

- 挙動を変更しない
- API仕様を変更しない
- コンポーネントの責務を明確にする
- 重複コードを削減する
- 可読性を向上させる

大規模な変更を行う場合は、変更理由を説明してください。

---

# ドキュメント作成

Issue、Pull Request、READMEなどを作成する場合は、

- 簡潔
- 分かりやすい
- 日本語

で作成してください。

---

# 禁止事項

以下は禁止します。

- 勝手なライブラリ追加
- any の安易な利用
- 既存アーキテクチャの変更
- 不要なリファクタリング
- プロジェクトで利用されていない実装パターンの導入

---

# 不明点がある場合

ルールが不明な場合は推測して実装せず、

1. 類似実装を探す
2. 関連ドキュメントを確認する
3. 判断できない場合はユーザーへ確認する

ことを優先してください。
